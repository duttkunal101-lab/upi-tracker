# Axis Bank — Agentic Credit-Card Onboarding (prototype)

A standalone, on-brand **agentic onboarding journey** for new Axis Bank credit-card customers — an AI
assistant (“Aria”) recommends the right card, auto-fills KYC from DigiLocker/CKYC, runs a consented
credit & income check, underwrites in real time, and issues an instant virtual card. Built to map a
**minimum-click, RBI/DPDP-aligned** onboarding end-to-end.

> This is a **separate project** from the CardWise app at the repository root — nothing here touches
> it. See **`BLUEPRINT.md`** for the full journey map, integrations, data points and compliance map.

## What you’re looking at

- A clickable, end-to-end journey (8 stages) with an autonomous **“agent at work”** execution UX,
  driven from just a phone number — Aria links DigiLocker in the background and auto-fills everything
  (PAN included), so the customer mostly confirms.
- **Trust at every automated step:** a reassurance row + a “why this is safe / what I do with your
  data” explainer on each sensitive step (KYC, eligibility, signing).
- A customer-facing **“Track my application”** drawer (the 📋 Track button) — a live status, a
  verification checklist (mobile → PAN → address → bureau → income → limit → issued → delivery), and
  the customer’s own (masked) details on file.
- **Multiple KYC paths, all demonstrable:** DigiLocker auto-fetch (default), Aadhaar-OTP e-KYC,
  document upload + **OCR**, and a well-planned **Video-KYC (V-CIP)** screen (officer, timings,
  what-to-keep-ready, what the officer does, schedule-for-later).
- **Credit-limit clarity:** a “how I arrived at your limit” breakdown (CIBIL band × verified income,
  kept within a safe FOIR) plus full card details & eligibility on the offer.
- **Gamification:** points, milestone badges, “card-school” education while the agent works, confetti
  and a scratch-to-reveal welcome reward.
- An always-on **co-pilot** chat, **drop-off nudges**, **exit-intent save-&-resume**, and a
  **“Behind the scenes”** blueprint drawer that exposes the integrations, data and RBI rules per step.
- The regulated rails (UIDAI, DigiLocker, CKYC, CIBIL, Account Aggregator, NPCI, eSign, issuance) are
  **simulated with clearly-labelled mock data** — a web prototype can’t touch the real systems.

## Structure

This folder is **self-contained** — it can be deployed as its own Vercel project (Root Directory
= `axis`), independently of the CardWise app at the repo root.

```
axis/
├── index.html        # app shell + all stage containers, co-pilot, tracker + blueprint drawers
├── styles.css        # Axis-branded (burgundy, light) styling — hand-written
├── js/
│   ├── config.js     # card catalogue, journey, integrations/data/regulatory registries, nudges
│   ├── integrations.js  # simulated rails (PAN/DigiLocker/CKYC/CIBIL/AA/penny-drop/eSign/issuance)
│   ├── agent.js      # AI co-pilot client + offline scripted fallback brain
│   └── app.js        # journey state-machine, agent overlay, nudges, save/resume, blueprint
├── api/
│   └── onboarding-agent.js   # serverless Claude concierge: recommend | ask | nudge (graceful fallback)
├── package.json      # standalone deps (@anthropic-ai/sdk) + `vercel dev` scripts
├── vercel.json       # function config (maxDuration) + no-cache headers
└── BLUEPRINT.md      # the written strategy (journey, RBI/DPDP, integrations, data, nudges)
```

Served as its own project, the app lives at the **domain root** (`https://<your-app>/`), not under
`/axis/`. It uses relative asset paths and an absolute `/api/onboarding-agent` call, so it works at
any base.

## Run it locally (from inside this folder)

**Static (built-in assistant, no AI key):** any static server works — the journey is fully
clickable and Aria uses her built-in scripted brain.

```bash
cd axis
python3 -m http.server 8000      # then open http://localhost:8000/   (root, not /axis/)
```

**Full (live AI agent):** run the serverless function locally with an Anthropic key so Aria’s
recommendations, Q&A and nudges are generated by Claude.

```bash
cd axis
npm install
npx vercel link                  # create/select a NEW project for the Axis app
npx vercel env pull .env.local   # or: export ANTHROPIC_API_KEY="sk-ant-..."
npx vercel dev                   # serves site + /api on http://localhost:3000/
```

## Deploy as its own Vercel project (independent of CardWise)

**CLI (simplest):**
```bash
cd axis
npx vercel            # first run creates a NEW project (its own domain) — accept the defaults
npx vercel env add ANTHROPIC_API_KEY     # add the key to THIS project
npx vercel --prod     # promote to production
```

**Dashboard (Git-integrated):** Vercel → *Add New Project* → import this repo → set
**Root Directory = `axis`** → add the `ANTHROPIC_API_KEY` env var → Deploy. Every push that touches
`axis/` redeploys this project; CardWise (root) stays a separate project.

> Because it’s a **separate project**, it has its own domain, env vars and deploys — the
> `ANTHROPIC_API_KEY` from the CardWise project does **not** carry over, so add it here (the site
> still runs without it, just with the scripted fallback).

## Configuration

| Env var | Purpose |
|---|---|
| `ANTHROPIC_API_KEY` | Enables the live Claude agent. **Without it the site still works** — Aria falls back to the built-in scripted assistant. Set a spend limit on the key; the endpoint is public once deployed. |
| `ONBOARDING_AGENT_MODEL` | Override the model (default `claude-opus-4-8`). |

To re-brand: edit the palette in `styles.css` (`:root`) and `brand` in `js/config.js`. To change the
card line-up, journey, integrations or compliance copy, edit `js/config.js` — the UI and the blueprint
drawer both render from it.

## Disclaimer

Working prototype for demonstration, **not production issuance**. Identity, bureau, Account-Aggregator
